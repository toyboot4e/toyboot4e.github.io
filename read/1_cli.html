Shell, Vim, Git に関する本をリストしました。

1.1. 新しい Linux の教科書

書名 新しいLinuxの教科書

著者

三宅 英明、大角 祐

出版日

2015/06/06

ページ数

440

Linux = Unix シェル (bash) とした入門書です。相対パスや絶対パス、環境変数など、 基礎知識のバックアップ を丁寧にやってくれる本でした。

内容は、ネット検索で手に入るものと言えますが、一冊の本として仕上げられており、ネットの断片的な情報よりも遥かに贅沢です。コラムも面白く、著者の心遣いを満喫できます。

初期投資に良い本でした。投資がスカになるのでは、という懸念はよくありますが、この本は違いました。

1.2. シェルプログラミング実用テクニック

書名 シェルプログラミング実用テクニック

著者

上田隆一

出版日

2015/05/15

ページ数

416

sed, grep, awk など、いわゆる Unix filter で one-liner program を書き、様々な作業を行います。この本のおかげで、僕はシェルを常用するようになりました。

文字列処理さえできれば、シェルは 8 割方マスターできたと言って良さそうです。その観点では、二章までが基本、三〜八章は応用となります。基本の中にエッセンスのクリームを漏れ出んばかりに押し込んでくれているので、応用部分は消化試合のように思いました。

濃い内容に、丁寧な説明が着いています。文章量が多く、初見では噛み砕くのが難しかったです。文章すべてがしっくりと来て、うんうんそうだね……と言えるようになったとき、本の真価が分かります。

読後は、胸を張って言って良いと思います。 コマンドライン チョットデキル 。……言い過ぎました。

1.3. フルスクラッチから1日でCMSを作る シェルスクリプト高速開発手法入門 改訂2版

書名 フルスクラッチから1日でCMSを作る シェルスクリプト高速開発手法入門 改訂2版

著者

上田隆一

出版日

2019/06/28

ページ数

304

ブログ用 CMS の自作をテーマにした本です。 markdown で書いた記事を GitHub に push すると、サーバが自動的に記事を更新する、という仕組みを整えます。

この本は、新たなシェルの入門書として作られたそうです。そして、 CMS の作成という『面白そうな』テーマが選ばれています。しかし、少なめの説明でどんどんシェルを使うため、この本が入門者向きであるとは思えません。

シェルの入門者が読みたいのは、『シェルプログラミング実用テクニック』の方ではないのかと思います。そして、そちらを読めば、この本を読む必要は無くなります。意図に反して、 マニアックな本 になったのではないかと捉えます [1] 。

2019 年に執筆された本だけあって、これ以上は無いと思うほど良い文で、シェルについて綴ってあります。 プロの物書きの文章 といった印象で、買って良かったと思いました。

通信には、 CGI という規格のおかげで、 stdin/stdout を使った普通のシェルスクリプトを使用できるようです。結局、 html の生成に最も力を使います。 CMS を作るのに必要な知識とは、 html/css の知識のようでした [2] 。

1.5. 独習Git

書名 独習Git

著者

リック・ウマリ (著), 吉川邦夫 (翻訳)

出版日

2016/2/26 (和書)

ページ数

384

ギター奏者のチュートリアルのように、『ゆっくりと演奏する』ことを心がけてGitを解説する、というような前書きの本でした。 モチベーションに良かった です。

working directory, index などの基本的な概念を導入し、 commit や merge ができるようになります。本の構成上、 .gitconfig は付録のような位置にあるので、早めに読んで、コマンドを便利にすると良いと思います。

merge, rebase については、合わせて 怖くないGit というスライドを見ると、理解が深まりました。それ以上の細かいコマンドは、必要に応じて調べたら良いと思います。

ちなみに、僕の .gitconfig の解説が ここ にあります。 Git に慣れていない人こそ、こういうのが欲しいのではないかと思って書きました。
