Dev 2, 2020

1. Text editors

evil-emacs-2020 を公開しました。空の ~/.emacs.d を 1 から設定した repository です。 Qiita の Emacs Advent Calendar 2020 向けに作成しました。

ここでは、各種エディタを振り返っていこうと思います。

1.1. VSCode

改めて、 VSCode は良いエディタだと思いました。 特に ターミナルペイン が良くて、他のエディタには匹敵するものが無いと思います。 VSCodeVim もよくできており、『エディタは VSCode だけ』というのも良いと思います。

1.2. Sublime

Sublime は センスの良いエディタ です。そして Sublime 以上に 動作が軽快 なエディタを知りません。しかし、拡張に不具合があって僕はダメでした。 Sublime 4 に期待しています。

1.3. NeoVim

以前の僕は、 NeoVim のユーザでした。 VSCode ユーザの人には、まず NeoVim に入門するのがお勧めです。 coc.nvim を使えば、 NeoVim はほぼ VSCode となります。プラグインの導入も、 vim-plug のおかげで楽々です。

1.4. Emacs

NeoVim から Emacs への移行は、一筋縄とは行きませんでした。 Evil の設定自体が大変なのと、 Emacs と Vim の違いも大きかったです。コツを 100 ~ 300 個くらい集めないと移行できません。そのため今回は、 evil-emacs-2020 に全コミットを投稿しました。

Distributions

Emacs の設定としては、 Spacemacs や Doom Emacs が有名です。これらも Evil (Vim のエミュレーション) を利用しており、 Vim の操作性が再現されています。しかし、 Vim との 細かな違いは、自分で修正する必要があります 。したがって、 distribution を使う場合も、 evil-emacs-2020 は役に立つと思います。

Evil Emacs

色々と試しましたが、 エディタ間に圧倒的な違いは無かった と思います。エディタの移行に生産性はありませんし、やりたい人だけがやればいいかなという感じです。僕の場合は、隣の芝生が青かったのでやってみました。得るものがあったので、第三第四の Vim が出てこないかなと期待しています。

kakoune は合いませんでした。 onivim とか moe はどうなんでしょうか。

Vanila Emacs

設定無しの Emacs も試してみました。特に面白かったのが、 スクロールが半ページ単位 (いわゆる type-writer scroll) だった ことです。 Vim ほどスクロール機能が充実していないため、この選択なのだと思いますが、センスの良さを感じました。

自分が Evil Emacs を使う場合は、 Vim のバインドで事足りるため、結局 1 行単位のスクロールに戻してしまいましたが……。
