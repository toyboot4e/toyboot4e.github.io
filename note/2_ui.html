1. About

標準的な UI は込み入っていていて、ユーザの画面を狭くしているという話です。

Qiita で書いた アプリのタイトルバーを隠す話 の物語的な補足となります。

1.1. Background

macOS では、アプリに リボン がありません。つまり、アプリのタイトルバーには、『ファイル』や『編集』のようなボタンが付いていません。代わりに、画面上部のメニューバーに、今フォーカスしているアプリのボタンが表示されます。多数ウィンドウのリボンが 1 つのメニューバーに集約されるため、画面範囲を節約できます [1] 。

この方式には批判もあって、大画面モニタ のユーザには、アプリ毎にボタンが付いていた方が便利だという意見もあります。しかし、リボンの機能はそう使わない上に、メニューバーはキーボード操作が可能です。したがって、メニューバーは、 余分な UI を減らしつつ、 機能へのアクセス を損なわない、優れた発明だと思います。

この体験に味を占め、同種の改善を求めてみました。特にラップトップ機では重要だと思います。

2. Using minimum UI

よく使うアプリから、余分な表示を減らしていきます。削るべき部分は、ウィンドウ上部に集まっていたと思います。

3. Wrapping up

ナビゲーションが UI の重要ポイントだと思います。キーボードのヘビーユーザになった後では、標準 UI にはかなりの無駄が見つかりました。しかし、 ユーザがそこまで進歩することを開発者が考慮してくれるわけではない ので、1 つ 1 つ UI を変更するしかないようです。このとき、好みの設定に移行するのが、 1 コマンドを走らせるくらい簡単だったら 助かったと思います [2] 。

改めて、 UI は手に負えないと思いました。特にウィンドウ・タブ・インスタンスの管理に関しては、どのエディタもどの window manager も、理想的ではないと思います。しかし、妥協を可能にする方向性は存在し、それは外部モニタを使うことです。 広い画面を持っていたなら 、画面の無駄はさほど問題になりませんし、今とは違った視点で UI に注目しただろうと思います [3] 。
